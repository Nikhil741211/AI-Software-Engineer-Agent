complete project workflow

This dashboard is structured according to the project statement: webhook integration, codebase exploration, fix generation, Docker testing, reasoning trace, cost guard, and human approval.

01

GitHub Webhook

New issue or label assignment triggers the FastAPI backend automatically.

02

Codebase Exploration

Agent scans files, searches relevant code, parses AST, and checks imports.

03

AI Fix + Tests

LLM generates a targeted function-level patch and validates it with tests.

04

Approval + PR

PR is created only after a human approves the generated change.

architecture

FastAPI, Redis, RQ Worker, Groq LLM, Docker, PostgreSQL, GitHub API, and frontend dashboard work together as a production-style automation pipeline.

01

FastAPI Backend

Receives GitHub webhook events and exposes approval, reject, diff, metrics, and issue APIs.

02

Redis + Worker

Queues GitHub issues and processes fixes asynchronously in the background.

03

AI Agent Core

Uses RAG search, AST parsing, function patching, cost guard, and loop detection.

04

Docker Sandbox

Executes tests safely and isolates code execution from the host system.
