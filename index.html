01

GitHub Webhook

New GitHub issue or label assignment triggers the agent automatically.

02

Codebase Exploration

Agent traverses files, searches code, parses AST, and identifies the root cause.

03

Fix Generation

Agent writes code fixes and generates tests covering the reported bug.

04

Human Approval

Pull request opens only after human reviews the reasoning and approves.

project architecture

The system connects frontend, FastAPI backend, Redis queue, AI agent worker, Docker sandbox, PostgreSQL, and GitHub pull request workflow.

01

Frontend UI

Issue queue, reasoning trace, diff viewer, approval interface, and metrics dashboard.

02

Backend API

FastAPI receives GitHub webhooks and stores jobs, traces, costs, and issue history.

03

Agent Worker

LangGraph-style worker uses read_file, search_code, write_file, and run_tests tools.

04

Safe Runtime

Docker sandbox blocks network access and runs tests with CPU and memory limits.
